- 训练流程：损失函数、优化器、学习率、梯度更新。
- 推理流程：自回归生成、温度采样、Top-k、Top-p。
- 工程支撑：PyTorch、CUDA、混合精度、模型保存与加载。

## 大模型基础概念

### 什么是语言模型

{IMAGE:5}

语言模型的任务是估计一个文本序列出现的概率。假设一句话由 token 序列组成：

$$
x = (x_1, x_2, \dots, x_T)
$$

语言模型要学习联合概率：

$$
P(x_1, x_2, \dots, x_T)
$$

根据链式法则，这个概率可以拆成：

$$
P(x_1, x_2, \dots, x_T) = \prod_{t=1}^{T} P(x_t \mid x_1, x_2, \dots, x_{t-1})
$$

也就是说，模型每一步都在做“根据前面的内容预测下一个 token”的任务。

{KNOWLEDGE}大语言模型中的“大”通常体现在参数规模、训练数据规模和计算资源规模上。但从学习角度，小模型和大模型共享同一套基本原理：token 输入、Transformer 编码、概率输出、自回归生成。{/KNOWLEDGE}

在 MiniMind 中，我们关心的不是 GPT-4、Claude、Gemini 这类超大模型的工业规模，而是如何用一个可运行、可训练、可解释的小模型复现语言模型的基本流程。

本节小结：语言模型的本质是条件概率建模，训练目标是让模型更准确地预测序列中的下一个 token。

## Token、Embedding 与上下文

### Token 是模型看到的基本单位

{IMAGE:6}

自然语言不能直接输入神经网络，必须先被转换成数字。分词器会把文本切分成 token，然后将 token 映射成整数 ID。例如：

```python
text = "你好，MiniMind"
tokens = ["你", "好", "，", "Mini", "Mind"]
ids = [102, 305, 11, 4210, 4211]
```

模型真正接收的是 `ids`，而不是原始字符串。

{WARNING}很多初学者会误以为模型直接理解汉字、单词或句子。实际上，模型处理的是 token ID，再通过 Embedding 层把离散 ID 转成连续向量。{/WARNING}

### Embedding 的作用

Embedding 层可以理解为一张查表矩阵。假设词表大小为 $V$，隐藏维度为 $d$，那么 Embedding 矩阵为：

$$
E \in \mathbb{R}^{V \times d}
$$

每个 token ID 会查到一个 $d$ 维向量：

$$
h_i = E[x_i]
$$

这些向量就是后续 Transformer 模块处理的输入。

```python
import torch
import torch.nn as nn

vocab_size = 5000
hidden_size = 256

embedding = nn.Embedding(vocab_size, hidden_size)

# batch=2, seq_len=4
input_ids = torch.tensor([
    [1, 23, 456, 78],
    [9, 10, 11, 12]
])

hidden_states = embedding(input_ids)

print(hidden_states.shape)  # torch.Size([2, 4, 256])
```

本节小结：Token 是文本的离散表示，Embedding 是离散 token 到连续向量空间的桥梁。

## Transformer 与 Attention

### Transformer 是大模型的核心结构

{IMAGE:7}

现代大语言模型大多基于 Transformer Decoder 架构。MiniMind 课程中也会围绕这一结构展开。Transformer 的关键组件包括：

- Token Embedding：表示输入 token。
- Position Embedding 或 RoPE：表示位置信息。
- Multi-Head Self-Attention：建模上下文依赖。
- Feed Forward Network：进行非线性特征变换。
- LayerNorm：稳定训练。
- Residual Connection：缓解深层网络训练困难。

### Self-Attention 的直观理解

Self-Attention 的作用是让序列中每个位置都能“看见”前面的相关信息。对于一个 token，它会生成三个向量：

$$
Q = XW_Q,\quad K = XW_K,\quad V = XW_V
$$

其中：

- $Q$：Query，表示当前位置想找什么信息。
- $K$：Key，表示每个位置能提供什么信息。
- $V$：Value，表示每个位置实际携带的信息。

注意力权重计算公式为：

$$
Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

其中 $\sqrt{d_k}$ 用来缩放点积结果，避免数值过大导致 softmax 梯度不稳定。

{IMPORTANT}Attention 的核心思想是：不是所有上下文 token 都同等重要，模型会动态计算每个 token 对当前预测的贡献权重。{/IMPORTANT}

本节小结：Transformer 通过 Self-Attention 建立长距离依赖，是现代语言模型的基础架构。

## 自回归生成与因果掩码

{IMAGE:8}

大语言模型通常采用自回归方式生成文本。给定提示词：

```text
大模型的核心能力是
```

模型先预测下一个 token，例如“语言”；然后把“语言”拼回输入，再继续预测下一个 token，如此循环。

```python
# 伪代码：自回归生成
for step in range(max_new_tokens):
    logits = model(input_ids)          # 得到每个位置的预测分布
    next_logits = logits[:, -1, :]     # 只取最后一个位置
    next_id = sample(next_logits)      # 采样或贪心选择
    input_ids = torch.cat([input_ids, next_id], dim=1)
```

### 因果掩码

训练 Decoder-only 模型时，第 $t$ 个位置只能看到 $1$ 到 $t$ 的 token，不能提前看到未来答案。否则模型会作弊。

因果掩码通常是一个上三角矩阵：

$$
M_{ij} =
\begin{cases}
0, & j \le i \\
-\infty, & j > i
\end{cases}
$$

经过 softmax 后，未来位置的注意力权重会变成 0。

{WARNING}如果没有因果掩码，训练 loss 可能看起来很低，但模型实际生成能力会很差，因为它在训练时偷看了未来 token。{/WARNING}

本节小结：自回归生成依赖“逐 token 预测”，因果掩码保证训练和推理任务一致。

## MiniMind 技术栈概览

{IMAGE:9}

MiniMind 的技术栈围绕 PyTorch 和大模型训练流程展开。课程中的关键技术包括：

### PyTorch

PyTorch 是深度学习模型实现的主要框架。它提供：

- `torch.Tensor`：张量计算。
- `torch.nn.Module`：模型模块封装。
- `autograd`：自动求导。
- `torch.optim`：优化器。
- CUDA 支持：GPU 加速训练。

```python
import torch
import torch.nn as nn

class TinyLinearModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(4, 2)

    def forward(self, x):
        return self.linear(x)

model = TinyLinearModel()
x = torch.randn(3, 4)
y = model(x)

print(y.shape)  # torch.Size([3, 2])
```

### 训练核心流程

{IMAGE:10}

一个标准训练循环包括：

1. 前向传播：模型根据输入计算输出。
2. 损失计算：比较预测和目标。
3. 反向传播：计算梯度。
4. 参数更新：优化器更新权重。
5. 清空梯度：准备下一步训练。

```python
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)
loss_fn = nn.CrossEntropyLoss()

for input_ids, labels in dataloader:
    logits = model(input_ids)

    # logits: [batch, seq_len, vocab_size]
    # labels: [batch, seq_len]
    loss = loss_fn(
        logits.view(-1, logits.size(-1)),
        labels.view(-1)
    )

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

本节小结：MiniMind 的主要技术栈是 PyTorch，学习重点是把语言模型训练流程逐步拆成可理解、可运行的代码。

## 损失函数与训练目标

{IMAGE:11}

语言模型训练通常使用交叉熵损失。对于每个位置，模型输出一个词表大小的 logits 向量，经过 softmax 得到概率分布：

$$
p_i = \frac{e^{z_i}}{\sum_j e^{z_j}}
$$

如果真实 token 是 $y$，交叉熵为：

$$
L = -\log p_y
$$

整个序列的训练损失通常是所有 token 位置损失的平均值：

$$
L_{seq} = \frac{1}{T}\sum_{t=1}^{T} -\log P(x_t \mid x_{<t})
$$

{KNOWLEDGE}训练语言模型时，输入和标签通常是错位的。例如输入是 `[x1, x2, x3]`，标签是 `[x2, x3, x4]`，模型学习的是“看到当前位置及之前内容后预测下一个 token”。{/KNOWLEDGE}

本节小结：大模型训练目标是最小化下一个 token 的预测误差，交叉熵是最常用的损失函数。

## MiniMind 学习路线

{IMAGE:1}

MiniMind 课程通常会按照从基础到系统的顺序推进：

### 1. 准备数据

{IMAGE:2}

数据是模型能力的来源。即使模型结构正确，如果数据质量差，模型也难以学到稳定能力。数据处理包括：

- 文本收集。
- 清洗乱码、重复、异常内容。
- 构建训练样本。
- 按固定长度切分 token 序列。
- 制作 batch。

### 2. 构建分词器

{IMAGE:12}

分词器决定文本如何变成 token。常见方案包括：

- 字符级分词：简单，但序列较长。
- WordPiece / BPE：常用于现代 NLP。
- SentencePiece：适合多语言和无空格语言。

### 3. 实现模型结构

{IMAGE:13}

模型结构包括 Embedding、Attention、MLP、LayerNorm 和输出头。输出头通常把隐藏状态映射回词表维度：

$$
logits = HW_{vocab}
$$

其中：

- $H \in \mathbb{R}^{B \times T \times d}$
- $W_{vocab} \in \mathbb{R}^{d \times V}$
- $logits \in \mathbb{R}^{B \times T \times V}$

### 4. 训练与验证

{IMAGE:14}

训练过程中要关注：

- loss 是否下降。
- 学习率是否合适。
- batch size 是否稳定。
- 是否出现梯度爆炸。
- 生成样例是否逐渐变好。

### 5. 推理与生成

{IMAGE:15}

推理阶段重点是采样策略。常见方法包括：

- Greedy Search：每次选概率最高的 token。
- Temperature Sampling：调节随机性。
- Top-k Sampling：只从前 k 个 token 中采样。
- Top-p Sampling：从累计概率达到 p 的候选集合中采样。

```python
def apply_temperature(logits, temperature=1.0):
    # temperature 越低，输出越确定；越高，随机性越强
    return logits / temperature
```

本节小结：MiniMind 的学习路线覆盖数据、分词、模型、训练、推理，是一个完整的大模型工程闭环。

## 参数规模、算力与小模型价值

{IMAGE:16}

大模型能力通常随参数规模、数据规模、训练算力提升而增强。但对于学习而言，直接训练超大模型并不现实。MiniMind 的价值在于用较小参数量复现核心结构。

模型参数量大致由以下部分组成：

- Embedding 参数：$V \times d$
- Attention 参数：$W_Q, W_K, W_V, W_O$
- MLP 参数：通常约为 $d \times 4d$ 和 $4d \times d$
- LayerNorm 参数：较少
- 输出层参数：$d \times V$

如果隐藏维度为 $d$，层数为 $L$，词表大小为 $V$，那么参数量会随 $L$、$d$、$V$ 快速增长。

{IMPORTANT}小模型的价值不是替代工业大模型，而是帮助学习者完整理解模型内部机制，并能在有限设备上完成实验闭环。{/IMPORTANT}

本节小结：MiniMind 通过小参数模型降低学习门槛，让大模型原理可以被亲手验证。

## 常见误区

{IMAGE:17}

### 误区一：只会调 API 就等于懂大模型

调用 API 能完成应用开发，但不代表理解模型原理。MiniMind 课程更关注底层实现，包括张量形状、训练目标、梯度更新和生成机制。

### 误区二：模型会真正“理解”语言

从技术实现看，语言模型是在高维向量空间中学习统计规律和上下文映射。它表现出理解能力，但内部仍是概率预测系统。

### 误区三：loss 越低模型越好

训练 loss 下降很重要，但不能单独代表模型质量。还需要看验证集 loss、生成质量、是否过拟合、是否重复输出、是否胡言乱语。

### 误区四：只看模型结构，不看数据

数据质量对模型效果影响极大。对于小模型尤其如此，低质量数据会迅速放大模型缺陷。

{WARNING}学习大模型时，不要只盯着 Transformer 结构。数据、训练策略、分词器和推理采样同样决定最终效果。{/WARNING}

本节小结：理解大模型需要系统视角，不能只看 API、结构或 loss 中的某一项。
