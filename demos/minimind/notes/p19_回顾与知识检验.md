## 第二部分：各模块详细回顾

### 2.1 Embedding与位置编码

{IMAGE:5}

#### 2.1.1 Token Embedding

Token Embedding将离散的token ID映射为连续的向量空间：

$$h_i = E \cdot x_i$$

其中 $E \in \mathbb{R}^{V \times D}$ 是嵌入矩阵，$V$ 为词表大小，$D$ 为隐藏层维度。

```python
class TokenEmbedding(nn.Module):
    def __init__(self, vocab_size, d_model):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, d_model)
    
    def forward(self, x):
        # x: [batch_size, seq_len] -> [batch_size, seq_len, d_model]
        return self.embedding(x) * math.sqrt(self.d_model)
```

#### 2.1.2 位置编码（Positional Encoding）

MiniMind使用旋转位置编码（RoPE），这是一种相对位置编码方案：

$$f_q(x_m, m) = W_q \cdot R_m^{\Theta,d}$$

{KNOWLEDGE}旋转位置编码通过在 Query 和 Key 向量上应用旋转矩阵来实现位置信息的注入，其优势在于可以处理任意长度的序列，并且具有良好的外推性。{/KNOWLEDGE}

```python
def precompute_freqs_cis(dim, end, theta=10000.0):
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2).float() / dim))
    t = torch.arange(end)
    freqs = torch.outer(t, freqs)
    return torch.polar(torch.ones_like(freqs), freqs)
```

### 2.2 Transformer Decoder Block

{IMAGE:6}

每个Decoder Block包含两个核心子层：

#### 2.2.1 多头自注意力（Multi-Head Self-Attention）

{IMAGE:7}

自注意力的核心计算：

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}} \cdot \text{mask}\right)V$$

```python
class SelfAttention(nn.Module):
    def __init__(self, n_heads, d_model):
        super().__init__()
        self.n_heads = n_heads
        self.head_dim = d_model // n_heads
        
        self.wq = nn.Linear(d_model, d_model)
        self.wk = nn.Linear(d_model, d_model)
        self.wv = nn.Linear(d_model, d_model)
        self.dense = nn.Linear(d_model, d_model)
        
    def apply_rotary_emb(self, x, freqs_cis):
        x_complex = torch.view_as_complex(x.float().reshape(*x.shape[:-1], -1, 2))
        x_rotated = x_complex * freqs_cis
        return torch.view_as_real(x_rotated).flatten(-2).type_as(x)
```

{WARNING}易错点：在实现RoPE时，必须确保Query和Key向量先进行分头处理，再应用旋转矩阵，而不是先旋转再分头。{/WARNING}

#### 2.2.2 前馈神经网络（FFN）

{IMAGE:8}

FFN采用SwiGLU激活函数：

$$\text{FFN}(x) = \text{SwiGLU}(W_1 x) \cdot W_3 x \cdot W_2$$

```python
class FeedForward(nn.Module):
    def __init__(self, d_model, intermediate_size):
        super().__init__()
        self.gate_proj = nn.Linear(d_model, intermediate_size, bias=False)
        self.up_proj = nn.Linear(d_model, intermediate_size, bias=False)
        self.down_proj = nn.Linear(intermediate_size, d_model, bias=False)
        self.act_fn = nn.SiLU()
        
    def forward(self, x):
        return self.down_proj(self.act_fn(self.gate_proj(x)) * self.up_proj(x))
```

### 2.3 LayerNorm与残差连接

{IMAGE:9}

每个子层都包含残差连接和层归一化：

$$\text{Output} = \text{LayerNorm}(x + \text{SubLayer}(x))$$

{IMPORTANT}核心概念：残差连接确保了深层网络的梯度流动，而LayerNorm则稳定了训练过程。二者配合使得模型能够堆叠数十层而保持稳定训练。{/IMPORTANT}

```python
class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))
    
    def forward(self, x):
        norm = x.pow(2).mean(-1, keepdim=True).rsqrt()
        return x * norm * self.weight
```

## 第三部分：核心概念知识检验

### 3.1 检验题目一：架构理解

{IMAGE:10}

**问题**: 为什么MiniMind选择Decoder-only架构而非Encoder-Decoder架构？

**答案要点**:
1. **训练效率**：Decoder-only只需进行一次前向传播和损失计算
2. **工程简洁**：无需维护Encoder-Decoder之间的交叉注意力
3. **涌现能力**：大规模预训练表明Decoder-only架构具有良好的缩放特性
4. **ChatGPT验证**：OpenAI的成功实践证明了此架构的可行性

{IMAGE:11}

### 3.2 检验题目二：注意力机制

**问题**: 解释因果掩码（Causal Mask）的作用，并说明其实现方式。

**答案**:

因果掩码确保每个token只能看到自己和之前的位置：

$$\text{Attention Score}_{i,j} = 
\begin{cases}
\text{原始分数} & \text{if } j \leq i \\
-\infty & \text{if } j > i
\end{cases}$$

```python
def apply_causal_mask(x):
    batch_size, n_heads, seq_len, _ = x.shape
    mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
    x = x.masked_fill(mask, float('-inf'))
    return x
```

### 3.3 检验题目三：位置编码

**问题**: RoPE相比绝对位置编码有哪些优势？

| 方面 | 绝对位置编码 | RoPE |
|------|-------------|------|
| 位置信息编码方式 | token位置 | token相对位置 |
| 外推能力 | 差，需训练更多位置 | 好，可处理更长序列 |
| 计算效率 | 较低 | 高，可预计算 |
| 与Attention结合 | 直接相加 | 旋转注入 |

## 第四部分：完整模型代码回顾

```python
class MiniMind(nn.Module):
    """
    MiniMind: 一个完整的LLM模型实现
    基于Transformer Decoder架构
    """
    def __init__(self, config):
        super().__init__()
        self.config = config
        
        # 词嵌入
        self.tok_embeddings = TokenEmbedding(config.vocab_size, config.d_model)
        
        # 位置编码（RoPE用）
        self.freqs_cis = precompute_freqs_cis(
            config.dim // config.n_heads, 
            config.max_seq_len
        )
        
        # N层Decoder Block
        self.layers = nn.ModuleList([
            TransformerBlock(config) for _ in range(config.n_layers)
        ])
        
        # 输出归一化
        self.norm = RMSNorm(config.dim)
        
        # 语言模型头部
        self.lm_head = nn.Linear(config.dim, config.vocab_size, bias=False)
    
    def forward(self, tokens):
        # 1. Token Embedding
        h = self.tok_embeddings(tokens)
        
        # 2. 添加RoPE位置信息（在Attention内部应用）
        # 3. 通过所有Transformer层
        for layer in self.layers:
            h = layer(h, self.freqs_cis)
        
        # 4. 最终归一化
        h = self.norm(h)
        
        # 5. 输出logits
        logits = self.lm_head(h)
        
        return logits
```

### 5.1 预训练流程

```
┌─────────────────────────────────────────────────────────────┐
│                      预训练阶段                               │
├─────────────────────────────────────────────────────────────┤
```

### 5.2 推理流程

```python
def generate(model, prompt, max_new_tokens=100):
    """自回归生成"""
    input_ids = tokenize(prompt)
    
    for _ in range(max_new_tokens):
        # 前向传播获取下一个token的logits
        logits = model(input_ids)
        next_token_logits = logits[:, -1, :]  # 只取最后一个位置
        
        # 采样（可使用贪心、top-k、top-p等策略）
        probs = F.softmax(next_token_logits / temperature, dim=-1)
        next_token = torch.multinomial(probs, num_samples=1)
        
        # 追加到输入
        input_ids = torch.cat([input_ids, next_token], dim=1)
        
        # 遇到EOS停止
        if next_token == EOS_TOKEN:
            break
    
    return detokenize(input_ids)
```

## 知识点总结

{IMPORTANT}本集核心要点{/IMPORTANT}

1. **架构选择**：MiniMind采用Decoder-only Transformer架构
2. **位置编码**：使用RoPE实现高效且具有良好外推性的位置编码
3. **注意力机制**：因果掩码确保自回归生成的正确性
4. **激活函数**：SwiGLU提供更好的非线性表达能力
5. **归一化**：RMSNorm相比LayerNorm计算效率更高
