## 一、PyTorch 张量：模型计算的基本单位

### 1. 什么是张量

在 PyTorch 中，`Tensor` 是最核心的数据结构。可以把它理解为多维数组：

- 标量：0 维张量，例如 `3.14`
- 向量：1 维张量，例如 `[1, 2, 3]`
- 矩阵：2 维张量，例如 `[[1, 2], [3, 4]]`
- 更高维张量：用于表示 batch、序列、特征、注意力头等

{IMAGE:1}

在深度学习中，数据通常不是单个数字，而是一批样本、一段序列、一组向量。例如语言模型中的输入通常可以表示为：

$$
X \in \mathbb{R}^{B \times T \times C}
$$

其中：

- $B$：batch size，批量大小
- $T$：sequence length，序列长度
- $C$：embedding dimension，词向量维度

{IMPORTANT}核心概念：在大模型代码中，理解每个张量的 shape 比记住某一行代码更重要。大多数 bug 都来自维度不匹配。{/IMPORTANT}

### 2. 创建张量

常见 PyTorch 张量创建方式如下：

```python
import torch

# 从 Python 列表创建张量
x = torch.tensor([[1, 2, 3], [4, 5, 6]])

# 创建全 0 张量
zeros = torch.zeros(2, 3)

# 创建全 1 张量
ones = torch.ones(2, 3)

# 创建随机张量
rand = torch.randn(2, 3)

print(x.shape)  # torch.Size([2, 3])
```

{IMAGE:2}

这里 `x.shape` 是非常关键的信息。它告诉我们张量有几维，每一维长度是多少。

### 3. 张量维度与语义

在写大模型时，维度本身并不只是数字，而是有语义的。

例如：

```python
x = torch.randn(4, 8, 16)
```

可以解释为：

- `4`：一次输入 4 个样本
- `8`：每个样本有 8 个 token
- `16`：每个 token 被表示成 16 维向量

也就是：

$$
x.shape = (B, T, C) = (4, 8, 16)
$$

{IMAGE:3}

{WARNING}易错点：不要只看张量是“三维的”，还要明确每一维表示什么。`(B, T, C)` 和 `(B, C, T)` 虽然都是三维，但语义完全不同，矩阵乘法和注意力计算都会受影响。{/WARNING}

本节小结：PyTorch 张量是模型数据流的基本单位，理解 shape 和每个维度的语义，是后续理解大模型代码的基础。

## 二、张量的基本操作

### 1. reshape：改变张量形状

`reshape` 用于改变张量形状，但元素总数必须保持一致。

```python
import torch

x = torch.arange(12)
print(x.shape)  # torch.Size([12])

y = x.reshape(3, 4)
print(y)
print(y.shape)  # torch.Size([3, 4])
```

数学上，reshape 并不改变数据值，只是改变观察数据的方式。

{IMAGE:4}

例如原始数据有 12 个元素，可以变成：

$$
12 = 3 \times 4 = 2 \times 6 = 2 \times 2 \times 3
$$

所以可以 reshape 为 `(3, 4)`、`(2, 6)`、`(2, 2, 3)` 等。

### 2. view 与 reshape 的区别

在很多代码中会看到：

```python
x = x.view(B, T, C)
```

`view` 和 `reshape` 都能改变形状，但 `view` 对内存连续性有要求，而 `reshape` 更灵活。

```python
x = torch.randn(2, 3, 4)
y = x.transpose(1, 2)

# y 可能不是连续内存
# y.view(2, 12) 可能报错
z = y.reshape(2, 12)
```

{WARNING}易错点：经过 `transpose`、`permute` 后，张量可能不再是连续内存。如果使用 `view` 报错，可以考虑先 `.contiguous()` 或直接使用 `reshape`。{/WARNING}

### 3. transpose / permute：交换维度

在注意力机制中，经常需要交换维度。

```python
x = torch.randn(2, 3, 4)

# 交换第 1 维和第 2 维
y = x.transpose(1, 2)

print(x.shape)  # torch.Size([2, 3, 4])
print(y.shape)  # torch.Size([2, 4, 3])
```

{IMAGE:5}

`transpose` 一次交换两个维度，`permute` 可以重新排列多个维度：

```python
x = torch.randn(2, 3, 4)

# 原维度: B, T, C
# 新维度: B, C, T
y = x.permute(0, 2, 1)

print(y.shape)  # torch.Size([2, 4, 3])
```

在多头注意力中，常见操作是把：

$$
(B, T, C)
$$

变成：

$$
(B, H, T, D)
$$

其中：

- $H$：注意力头数
- $D$：每个头的维度
- $C = H \times D$

{IMAGE:6}

本节小结：reshape、view、transpose、permute 是理解大模型张量变换的基础，尤其在多头注意力中会频繁出现。

## 三、矩阵乘法：神经网络的核心运算

### 1. 普通矩阵乘法

矩阵乘法要求左矩阵的列数等于右矩阵的行数。

如果：

$$
A \in \mathbb{R}^{m \times n}
$$

$$
B \in \mathbb{R}^{n \times p}
$$

那么：

$$
C = AB \in \mathbb{R}^{m \times p}
$$

{IMAGE:7}

具体计算方式是：

$$
C_{ij} = \sum_{k=1}^{n} A_{ik}B_{kj}
$$

也就是第一个矩阵的第 $i$ 行，与第二个矩阵的第 $j$ 列做点积。

### 2. PyTorch 中的矩阵乘法

```python
import torch

A = torch.randn(2, 3)
B = torch.randn(3, 4)

C = A @ B
# 等价于 C = torch.matmul(A, B)

print(C.shape)  # torch.Size([2, 4])
```

这里：

$$
(2, 3) @ (3, 4) = (2, 4)
$$

{IMAGE:8}

{IMPORTANT}核心概念：矩阵乘法不是逐元素相乘。逐元素相乘要求两个张量形状兼容，而矩阵乘法关注的是内维度是否相等。{/IMPORTANT}

### 3. 批量矩阵乘法

在深度学习中，通常不是只乘两个二维矩阵，而是对 batch 中的多个矩阵同时做乘法。

例如：

```python
A = torch.randn(5, 2, 3)
B = torch.randn(5, 3, 4)

C = torch.matmul(A, B)

print(C.shape)  # torch.Size([5, 2, 4])
```

这里相当于有 5 组矩阵，每组执行：

$$
(2, 3) @ (3, 4) = (2, 4)
$$

整体结果为：

$$
(5, 2, 4)
$$

{IMAGE:9}

### 4. 线性层与矩阵乘法

神经网络中的线性层本质上就是矩阵乘法加偏置：

$$
Y = XW + b
$$

其中：

- $X$：输入
- $W$：权重矩阵
- $b$：偏置
- $Y$：输出

```python
import torch
import torch.nn as nn

linear = nn.Linear(16, 32)

x = torch.randn(4, 8, 16)
y = linear(x)

print(y.shape)  # torch.Size([4, 8, 32])
```

这里 `nn.Linear(16, 32)` 会把最后一维从 16 映射到 32：

$$
(B, T, 16) \rightarrow (B, T, 32)
$$

{IMAGE:10}

{KNOWLEDGE}背景知识：PyTorch 的 `nn.Linear(in_features, out_features)` 作用在输入张量的最后一维上。前面的 batch 维和 sequence 维会被保留。{/KNOWLEDGE}

本节小结：矩阵乘法是线性层、注意力分数计算、输出映射等操作的基础，关键是看清楚相乘张量的最后两个维度。

## 四、注意力机制的直观理解

### 1. 为什么需要注意力

在语言模型中，一个 token 的含义往往依赖上下文。

例如句子：

> “我把苹果放进书包，因为它很重。”

这里“它”指什么，需要结合前文判断。注意力机制的目标就是：让模型在处理当前 token 时，能够根据相关性关注其他 token。

{IMAGE:11}

注意力机制可以理解为一个“信息检索”过程：

- Query：当前 token 想查什么
- Key：每个 token 提供的索引信息
- Value：每个 token 真正携带的内容信息

即：

$$
Attention(Q, K, V)
$$

### 2. Q、K、V 的来源

在 Transformer 中，输入 token embedding 会经过三个线性层，分别生成：

$$
Q = XW_Q
$$

$$
K = XW_K
$$

$$
V = XW_V
$$

其中 $X$ 是输入张量。

```python
import torch
import torch.nn as nn

B, T, C = 2, 4, 8

x = torch.randn(B, T, C)

q_proj = nn.Linear(C, C)
k_proj = nn.Linear(C, C)
v_proj = nn.Linear(C, C)

q = q_proj(x)
k = k_proj(x)
v = v_proj(x)

print(q.shape, k.shape, v.shape)
# torch.Size([2, 4, 8]) torch.Size([2, 4, 8]) torch.Size([2, 4, 8])
```

{IMAGE:12}

{IMPORTANT}核心概念：Q、K、V 不是原始输入本身，而是输入经过不同线性变换后得到的三组表示。它们来自同一个输入，但承担不同角色。{/IMPORTANT}

本节小结：注意力机制让每个 token 根据相关性从其他 token 中聚合信息，Q、K、V 分别表示查询、匹配键和值内容。

## 五、Scaled Dot-Product Attention 计算过程

### 1. 第一步：计算注意力分数

注意力分数由 Query 和 Key 的点积得到：

$$
S = QK^T
$$

如果：

$$
Q \in \mathbb{R}^{T \times C}
$$

$$
K \in \mathbb{R}^{T \times C}
$$

那么：

$$
K^T \in \mathbb{R}^{C \times T}
$$

所以：

$$
S = QK^T \in \mathbb{R}^{T \times T}
$$

{IMAGE:13}

$S_{ij}$ 表示第 $i$ 个 token 对第 $j$ 个 token 的关注程度。

```python
import torch

B, T, C = 2, 4, 8

q = torch.randn(B, T, C)
k = torch.randn(B, T, C)

scores = q @ k.transpose(-2, -1)

print(scores.shape)  # torch.Size([2, 4, 4])
```

这里：

$$
(B, T, C) @ (B, C, T) = (B, T, T)
$$

{IMAGE:14}

### 2. 第二步：缩放

直接点积时，如果维度 $C$ 较大，点积结果可能数值过大，导致 softmax 后分布过于尖锐，梯度不稳定。

因此使用缩放：

$$
S = \frac{QK^T}{\sqrt{d_k}}
$$

其中 $d_k$ 是 key/query 的维度。

```python
import math

scores = q @ k.transpose(-2, -1)
scores = scores / math.sqrt(C)
```

{IMAGE:15}

{KNOWLEDGE}背景知识：缩放因子 $\sqrt{d_k}$ 的作用是控制点积数值规模，让 softmax 的输入更稳定，从而帮助训练。{/KNOWLEDGE}

### 3. 第三步：softmax 得到权重

注意力分数经过 softmax 后，变成概率分布：

$$
A = softmax(S)
$$

每一行代表当前 token 对所有 token 的关注权重，并且每一行加起来等于 1。

```python
att = torch.softmax(scores, dim=-1)

print(att.shape)  # torch.Size([2, 4, 4])
```

{IMAGE:16}

如果某一行是：

$$
[2.0, 1.0, 0.1, -1.0]
$$

softmax 后可能变成类似：

$$
[0.65, 0.24, 0.10, 0.01]
$$

表示当前 token 更关注第一个位置。

{WARNING}易错点：softmax 的维度通常是 `dim=-1`，表示在最后一维上归一化。对于注意力分数 `(B, T, T)`，最后一维代表“当前 token 可以关注哪些 token”。{/WARNING}

### 4. 第四步：加权求和 Value

最后用注意力权重乘以 Value：

$$
O = AV
$$

如果：

$$
A \in \mathbb{R}^{T \times T}
$$

$$
V \in \mathbb{R}^{T \times C}
$$

那么：

$$
O \in \mathbb{R}^{T \times C}
$$

```python
out = att @ v

print(out.shape)  # torch.Size([2, 4, 8])
```

{IMAGE:17}

完整公式为：

$$
Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

这就是 Transformer 中最核心的注意力公式。

本节小结：Scaled Dot-Product Attention 分为四步：$QK^T$ 算相关性，除以 $\sqrt{d_k}$ 做缩放，softmax 得到权重，再乘以 $V$ 聚合信息。

## 六、因果注意力与 Mask

### 1. 自回归语言模型为什么需要 Mask

MiniMind 作为语言模型，需要根据前面的 token 预测下一个 token。训练时，虽然整段文本一次性输入模型，但模型在预测当前位置时不能看到未来 token。

例如序列：

$$
[x_1, x_2, x_3, x_4]
$$

当模型处理 $x_2$ 时，只能看 $x_1$ 和 $x_2$，不能看 $x_3$、$x_4$。

{IMAGE:18}

这就需要因果 Mask，也称 causal mask。

### 2. 下三角 Mask

因果注意力通常使用下三角矩阵：

$$
\begin{bmatrix}
1 & 0 & 0 & 0 \\
1 & 1 & 0 & 0 \\
1 & 1 & 1 & 0 \\
1 & 1 & 1 & 1
\end{bmatrix}
$$

其中：

- `1` 表示可以关注
- `0` 表示不能关注

{IMAGE:19}

在代码中可以这样构造：

```python
import torch

T = 4

mask = torch.tril(torch.ones(T, T))
print(mask)
```

输出形式类似：

```python
tensor([[1., 0., 0., 0.],
        [1., 1., 0., 0.],
        [1., 1., 1., 0.],
        [1., 1., 1., 1.]])
```

### 3. Mask 如何作用于注意力分数

Mask 通常在 softmax 之前施加。不能关注的位置被填成负无穷：

```python
scores = scores.masked_fill(mask == 0, float("-inf"))
att = torch.softmax(scores, dim=-1)
```

{IMAGE:20}

这样 softmax 后，被 mask 的位置权重会变成 0。

数学上可以理解为：

$$
softmax(-\infty) = 0
$$

{WARNING}易错点：Mask 必须加在 softmax 之前，而不是 softmax 之后。因为 softmax 是归一化操作，先 mask 才能保证不可见位置不会分到概率质量。{/WARNING}

本节小结：因果 Mask 保证语言模型不能偷看未来 token，是自回归生成任务中的关键约束。

## 七、多头注意力的基本思想

### 1. 为什么需要多头

单个注意力头只能在一个表示空间中计算相关性。多头注意力则把通道维度拆成多个子空间，让不同头学习不同类型的关系。

例如：

- 一个头关注语法关系
- 一个头关注实体指代
- 一个头关注局部相邻词
- 一个头关注长距离依赖

{IMAGE:21}

设输入维度为：

$$
C = 64
$$

头数为：

$$
H = 4
$$

则每个头的维度：

$$
D = \frac{C}{H} = 16
$$

### 2. 多头维度拆分

原始输入：

$$
X \in \mathbb{R}^{B \times T \times C}
$$

生成 Q、K、V 后，将最后一维拆成：

$$
(B, T, H, D)
$$

再转置为：

$$
(B, H, T, D)
$$

方便每个头独立做注意力计算。

```python
import torch
import torch.nn as nn
import math

B, T, C = 2, 4, 8
H = 2
D = C // H

x = torch.randn(B, T, C)

q_proj = nn.Linear(C, C)
k_proj = nn.Linear(C, C)
v_proj = nn.Linear(C, C)

q = q_proj(x).reshape(B, T, H, D).transpose(1, 2)
k = k_proj(x).reshape(B, T, H, D).transpose(1, 2)
v = v_proj(x).reshape(B, T, H, D).transpose(1, 2)

print(q.shape)  # torch.Size([2, 2, 4, 4])
```

{IMAGE:22}

此时：

$$
q, k, v \in \mathbb{R}^{B \times H \times T \times D}
$$

注意力分数为：

$$
scores = qk^T
$$

对应维度：

$$
(B, H, T, D) @ (B, H, D, T) = (B, H, T, T)
$$

### 3. 多头注意力代码示例

```python
import torch
import torch.nn as nn
import math

class SimpleCausalSelfAttention(nn.Module):
    def __init__(self, n_embd, n_head, block_size):
        super().__init__()

        assert n_embd % n_head == 0

        self.n_head = n_head
        self.head_dim = n_embd // n_head

        self.q_proj = nn.Linear(n_embd, n_embd)
        self.k_proj = nn.Linear(n_embd, n_embd)
        self.v_proj = nn.Linear(n_embd, n_embd)
        self.out_proj = nn.Linear(n_embd, n_embd)

        # 注册下三角 mask，形状为 (1, 1, T, T)，方便广播
        mask = torch.tril(torch.ones(block_size, block_size))
        self.register_buffer("mask", mask.view(1, 1, block_size, block_size))

    def forward(self, x):
        B, T, C = x.shape

        q = self.q_proj(x)
        k = self.k_proj(x)
        v = self.v_proj(x)

        # (B, T, C) -> (B, H, T, D)
        q = q.reshape(B, T, self.n_head, self.head_dim).transpose(1, 2)
        k = k.reshape(B, T, self.n_head, self.head_dim).transpose(1, 2)
        v = v.reshape(B, T, self.n_head, self.head_dim).transpose(1, 2)

        # 注意力分数: (B, H, T, T)
        scores = q @ k.transpose(-2, -1)
        scores = scores / math.sqrt(self.head_dim)

        # 只保留当前位置及之前的位置
        scores = scores.masked_fill(self.mask[:, :, :T, :T] == 0, float("-inf"))

        att = torch.softmax(scores, dim=-1)

        # 聚合 value: (B, H, T, D)
        out = att @ v

        # 合并多头: (B, H, T, D) -> (B, T, C)
        out = out.transpose(1, 2).reshape(B, T, C)

        return self.out_proj(out)
```

{IMAGE:23}

{IMPORTANT}核心概念：多头注意力不是把多个模型简单相加，而是把同一个 embedding 空间拆成多个 head，各自计算注意力，再拼接回原维度。{/IMPORTANT}

本节小结：多头注意力通过拆分通道维度，让模型在多个子空间中并行学习不同关系，是 Transformer 表达能力的重要来源。

## 八、从前置知识连接到 MiniMind

### 1. MiniMind 中会反复出现的结构

后续手写大模型时，会不断遇到以下操作：

```python
x = token_embedding(idx)
x = x + position_embedding
x = attention(x)
x = mlp(x)
logits = lm_head(x)
```

这些代码背后的核心依赖包括：

- embedding 查表得到 `(B, T, C)`
- linear 改变最后一维
- attention 计算 `(B, H, T, T)`
- mask 防止看到未来
- softmax 得到概率分布
- matmul 完成信息聚合

{IMAGE:25}

### 2. 重点维度流动

假设：

- batch size：$B$
- 序列长度：$T$
- 模型维度：$C$
- 注意力头数：$H$
- 单头维度：$D = C / H$

那么自注意力中的主要维度变化是：

$$
X: (B, T, C)
$$

$$
Q,K,V: (B, T, C)
$$

$$
Q,K,V \rightarrow (B, H, T, D)
$$

$$
QK^T: (B, H, T, T)
$$

$$
softmax(QK^T / \sqrt{D}): (B, H, T, T)
$$

$$
Attention \ Output: (B, H, T, D)
$$

## 九、本集核心公式汇总

### 1. 线性层

$$
Y = XW + b
$$

说明：将输入最后一维从 `in_features` 映射到 `out_features`。

### 2. 注意力分数

$$
S = QK^T
$$

说明：计算每个 token 对其他 token 的相关性。

### 3. 缩放注意力

$$
S = \frac{QK^T}{\sqrt{d_k}}
$$

说明：避免点积值过大导致 softmax 饱和。

### 4. 注意力输出

$$
Attention(Q,K,V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

说明：先计算关注权重，再对 Value 做加权求和。

### 5. 多头拆分

$$
C = H \times D
$$

说明：模型总维度等于头数乘以每个头的维度。

本节小结：掌握这些公式，就能读懂 Transformer 注意力模块的大部分核心代码。

## 十、关键收获

1. PyTorch 中张量是大模型计算的基本单位，必须关注 shape 和每个维度的语义。  
2. 矩阵乘法要求内维度匹配，是线性层和注意力机制的核心运算。  
3. 注意力机制通过 $QK^T$ 计算 token 之间的相关性，再用 softmax 权重聚合 $V$。  
4. 缩放因子 $\sqrt{d_k}$ 用于稳定 softmax 输入。  
5. 因果 Mask 防止模型在训练时看到未来 token，是自回归语言模型的必要条件。  
6. 多头注意力通过拆分通道维度，让模型在多个子空间中学习不同关系。  
7. 后续 MiniMind 的核心代码，本质上都是这些基础操作的组合。
