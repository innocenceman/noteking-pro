## 课程定位与本集目标

本集是 MiniMind 课程第 5/26 集，主题是“架构图解读”，核心任务是把 Transformer 架构图从输入到输出逐层拆开，理解每个模块在大语言模型中的位置、作用和数据流向。

{IMAGE:6}

Transformer 并不是一个单一算子，而是一套由多个基础模块堆叠起来的序列建模架构。对于 MiniMind 这种从零手敲大模型的课程来说，先读懂架构图非常关键，因为后续 PyTorch 代码基本就是把图中的每个方块翻译成模块类。

{IMPORTANT}本集的核心目标：看懂 Transformer 全图，知道 token 如何经过 Embedding、位置编码、多层 Transformer Block、注意力、前馈网络、归一化，最终变成词表概率分布。{/IMPORTANT}

本节小结：Transformer 架构图可以看作一条数据流水线，输入文本被编码成向量，经过多层特征变换，最后预测下一个 token。

## Transformer 总体结构

{IMAGE:1}

经典 Transformer 架构通常分为 Encoder 和 Decoder 两部分。但在大语言模型中，尤其是 GPT、LLaMA、MiniMind 这类自回归生成模型，通常使用的是 Decoder-only 架构。

也就是说，模型主要完成这样的任务：

给定前面的 token：

$$
x_1, x_2, ..., x_t
$$

预测下一个 token：

$$
P(x_{t+1} \mid x_1, x_2, ..., x_t)
$$

从图上看，完整流程大致是：

1. 输入 token ids
2. Token Embedding
3. Position Embedding 或 RoPE 位置编码
4. 多层 Transformer Block
5. 最后一层归一化
6. Linear 投影到词表维度
7. Softmax 得到概率
8. 选择或采样下一个 token

{IMAGE:2}

{KNOWLEDGE}在大模型训练中，模型并不是一次只预测一个 token，而是对整个序列并行预测。比如输入长度为 $T$，模型会同时预测每个位置的下一个 token。{/KNOWLEDGE}

本节小结：MiniMind 关注的是 Decoder-only Transformer，它通过自回归方式学习“根据上文预测下一个词”。

## 输入层：Token 与 Tokenizer

{IMAGE:3}

神经网络不能直接处理自然语言文本，所以第一步是把文本切成 token，再把 token 映射成整数 id。

例如：

```text
"我喜欢学习大模型"
```

可能被 tokenizer 切成：

```text
["我", "喜欢", "学习", "大", "模型"]
```

然后映射成 token ids：

```text
[128, 2048, 3521, 67, 912]
```

这些整数本身没有语义，真正有语义的是后续的 Embedding 向量。

在 PyTorch 中，输入通常是一个二维张量：

```python
# batch_size = B, sequence_length = T
input_ids.shape == (B, T)
```

例如：

```python
import torch

input_ids = torch.tensor([
    [128, 2048, 3521, 67, 912],
    [33, 809, 1001, 205, 0]
])

print(input_ids.shape)  # torch.Size([2, 5])
```

{WARNING}token id 只是词表中的编号，不是词向量。很多初学者会误以为 token id 越接近，语义越接近，这是错误的。语义关系是在 Embedding 空间中学习出来的。{/WARNING}

本节小结：文本先经过 tokenizer 转成 token ids，token ids 只是索引，后续需要 Embedding 才能进入神经网络计算。

## Token Embedding：从离散编号到连续向量

{IMAGE:7}

Embedding 层的作用是把每个 token id 映射成一个高维向量。假设词表大小为 $V$，模型隐藏维度为 $d$，则 Embedding 矩阵为：

$$
E \in \mathbb{R}^{V \times d}
$$

输入 token id $x_i$ 对应的向量就是矩阵中的一行：

$$
h_i = E[x_i]
$$

如果输入形状是：

$$
(B, T)
$$

经过 Embedding 后变成：

$$
(B, T, d)
$$

对应 PyTorch 代码：

```python
import torch
import torch.nn as nn

vocab_size = 6400
hidden_size = 512

tok_embedding = nn.Embedding(vocab_size, hidden_size)

input_ids = torch.randint(0, vocab_size, (2, 8))
x = tok_embedding(input_ids)

print(x.shape)  # torch.Size([2, 8, 512])
```

这里的 `512` 就是每个 token 的向量维度。模型训练过程中，Embedding 矩阵会不断更新，使语义相近的 token 在向量空间中更接近。

{IMPORTANT}Embedding 层本质上是一个可训练查表操作。它不是人工设计语义，而是通过语言模型训练自动学出语义表示。{/IMPORTANT}

本节小结：Embedding 把离散 token id 转成连续向量，是文本进入 Transformer 的第一道神经网络层。

## 位置编码：让模型知道顺序

{IMAGE:8}

Self-Attention 本身对输入顺序不敏感。如果只给模型一组 token 向量，它并不知道哪个 token 在前、哪个 token 在后。因此需要引入位置信息。

序列中同样的 token 在不同位置，含义可能不同。例如：

```text
我喜欢你
你喜欢我
```

这两个句子 token 类似，但顺序不同，语义不同。

位置编码的常见方式有：

1. 绝对位置编码
2. 可学习位置 Embedding
3. RoPE 旋转位置编码
4. ALiBi 等相对位置方法

经典 Transformer 使用正弦余弦位置编码：

$$
PE_{(pos, 2i)} = \sin \left(\frac{pos}{10000^{2i/d}}\right)
$$

$$
PE_{(pos, 2i+1)} = \cos \left(\frac{pos}{10000^{2i/d}}\right)
$$

其中：

- $pos$ 表示 token 在序列中的位置
- $i$ 表示向量维度索引
- $d$ 表示隐藏维度

MiniMind 这类现代 Decoder-only 模型通常更倾向使用 RoPE，因为它更适合长上下文建模。

{KNOWLEDGE}RoPE 不再简单地把位置向量加到 token embedding 上，而是通过旋转 Query 和 Key 向量，把相对位置信息编码进注意力计算中。{/KNOWLEDGE}

本节小结：位置编码解决 Transformer 不知道 token 顺序的问题，是序列建模不可缺少的部分。

## Transformer Block 总览

{IMAGE:9}

一个 Decoder-only Transformer 通常由多个相同结构的 Block 堆叠而成。每个 Block 大致包含：

1. LayerNorm 或 RMSNorm
2. Multi-Head Self-Attention
3. 残差连接
4. LayerNorm 或 RMSNorm
5. Feed Forward Network
6. 残差连接

可以抽象为：

$$
x = x + Attention(Norm(x))
$$

$$
x = x + FFN(Norm(x))
$$

这种结构称为 Pre-Norm，即归一化放在子层之前。现代大模型多数采用 Pre-Norm，因为训练更稳定。

对应伪代码：

```python
class TransformerBlock(nn.Module):
    def __init__(self, attention, feed_forward, norm1, norm2):
        super().__init__()
        self.attention = attention
        self.feed_forward = feed_forward
        self.norm1 = norm1
        self.norm2 = norm2

    def forward(self, x):
        # 注意力子层 + 残差连接
        x = x + self.attention(self.norm1(x))

        # 前馈网络子层 + 残差连接
        x = x + self.feed_forward(self.norm2(x))

        return x
```

{IMPORTANT}Transformer 的核心不是单个 Attention，而是 Attention、FFN、Norm、Residual 多个模块的组合。{/IMPORTANT}

本节小结：Transformer Block 是模型主体，多个 Block 堆叠后形成深层语义表示能力。

## 自注意力机制：让 token 互相看见

{IMAGE:10}

Self-Attention 的作用是让序列中的每个 token 根据上下文动态更新自己的表示。

对输入 $X$ 做三组线性变换，得到 Query、Key、Value：

$$
Q = XW_Q
$$

$$
K = XW_K
$$

$$
V = XW_V
$$

其中：

- Query 表示“我想找什么信息”
- Key 表示“我有什么信息可被匹配”
- Value 表示“如果被关注，我提供什么内容”

注意力权重计算公式：

$$
Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

这里除以 $\sqrt{d_k}$ 是为了避免点积值过大，导致 softmax 进入饱和区，梯度变小。

{IMAGE:11}

简单 PyTorch 版本：

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(q, k, v, mask=None):
    # q, k, v: (B, heads, T, head_dim)
    head_dim = q.size(-1)

    # 注意力分数: (B, heads, T, T)
    scores = q @ k.transpose(-2, -1) / (head_dim ** 0.5)

    # mask 用于屏蔽未来信息
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))

    weights = F.softmax(scores, dim=-1)

    # 加权求和得到输出
    output = weights @ v
    return output
```

{WARNING}Attention 权重不是固定词典关系，而是随输入上下文动态变化。同一个 token 在不同句子里关注的对象可能完全不同。{/WARNING}

本节小结：Self-Attention 通过 Query、Key、Value 计算 token 间关系，是 Transformer 的核心信息交互机制。

## 因果遮罩：防止偷看未来

{IMAGE:12}

在语言模型中，预测第 $t$ 个位置时，只能使用它之前的信息，不能看到未来 token。否则训练时模型会作弊。

例如训练句子：

```text
我 喜欢 学习 大 模型
```

当模型预测“学习”时，只能看见：

```text
我 喜欢
```

不能看见：

```text
大 模型
```

因此需要 causal mask。注意力矩阵中，每个位置只能关注自己和之前的位置。

矩阵形式如下：

$$
\begin{bmatrix}
1 & 0 & 0 & 0 \\
1 & 1 & 0 & 0 \\
1 & 1 & 1 & 0 \\
1 & 1 & 1 & 1
\end{bmatrix}
$$

代码示例：

```python
import torch

T = 4

# 下三角矩阵，允许当前位置看见自己和过去
mask = torch.tril(torch.ones(T, T))

print(mask)
```

在 attention scores 中，被 mask 的位置通常填入 $-\infty$：

```python
scores = scores.masked_fill(mask == 0, float("-inf"))
```

这样经过 softmax 后，未来位置的概率会变成 0。

{IMPORTANT}Decoder-only 语言模型必须使用因果遮罩，否则训练目标会被破坏，模型会通过未来 token 直接得到答案。{/IMPORTANT}

本节小结：Causal mask 保证模型只能利用历史上下文预测未来，是自回归语言模型成立的前提。

## 多头注意力：从多个角度看上下文

{IMAGE:13}

单头注意力只能在一个表示子空间里计算关系，而多头注意力把隐藏维度拆成多个 head，让模型从不同角度理解上下文。

假设隐藏维度为 $d$，头数为 $h$，则每个头的维度为：

$$
d_{head} = \frac{d}{h}
$$

例如：

$$
d = 512, \quad h = 8
$$

则：

$$
d_{head} = 64
$$

多头注意力过程：

1. 输入 $X$ 投影成 $Q, K, V$
2. 拆成多个 head
3. 每个 head 独立计算 attention
4. 拼接所有 head
5. 再通过输出线性层

代码结构示意：

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, hidden_size, num_heads):
        super().__init__()
        assert hidden_size % num_heads == 0

        self.hidden_size = hidden_size
        self.num_heads = num_heads
        self.head_dim = hidden_size // num_heads

        self.q_proj = nn.Linear(hidden_size, hidden_size)
        self.k_proj = nn.Linear(hidden_size, hidden_size)
        self.v_proj = nn.Linear(hidden_size, hidden_size)
        self.o_proj = nn.Linear(hidden_size, hidden_size)

    def forward(self, x, mask=None):
        B, T, C = x.shape

        q = self.q_proj(x)
        k = self.k_proj(x)
        v = self.v_proj(x)

        # (B, T, C) -> (B, heads, T, head_dim)
        q = q.view(B, T, self.num_heads, self.head_dim).transpose(1, 2)
        k = k.view(B, T, self.num_heads, self.head_dim).transpose(1, 2)
        v = v.view(B, T, self.num_heads, self.head_dim).transpose(1, 2)

        out = scaled_dot_product_attention(q, k, v, mask)

        # 拼回 hidden_size
        out = out.transpose(1, 2).contiguous().view(B, T, C)

        return self.o_proj(out)
```

{KNOWLEDGE}不同 head 不需要人为指定功能。训练后，有些 head 可能更关注局部语法，有些更关注长距离依赖，有些可能关注标点、格式或特殊 token。{/KNOWLEDGE}

本节小结：多头注意力让模型在多个子空间并行建模上下文关系，提高表达能力。

## 残差连接：让深层模型更容易训练

{IMAGE:14}

Transformer 中大量使用残差连接。残差连接的形式是：

$$
y = x + F(x)
$$

其中 $x$ 是原始输入，$F(x)$ 是子层输出。

它有两个重要作用：

1. 保留原始信息，避免特征在深层网络中被完全覆盖
2. 改善梯度传播，使深层网络更容易训练

在 Transformer Block 中，注意力层和 FFN 层后面都会接残差：

```python
x = x + attention(norm(x))
x = x + feed_forward(norm(x))
```

这意味着每一层不是从零重写表示，而是在原有表示上做增量更新。

{WARNING}残差连接要求输入和输出形状一致。如果 attention 或 FFN 输出维度和输入 hidden size 不一致，就不能直接相加。{/WARNING}

本节小结：残差连接是深层 Transformer 能稳定训练的重要机制，它让每层学习“修正量”而不是完整重建。

## 归一化层：稳定训练过程

{IMAGE:4}

Transformer 中常见归一化方式有 LayerNorm 和 RMSNorm。MiniMind 这类轻量实现常会使用 RMSNorm，因为它更简单、效率更高。

LayerNorm 公式：

$$
LayerNorm(x) = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta
$$

RMSNorm 公式：

$$
RMSNorm(x) = \frac{x}{\sqrt{mean(x^2) + \epsilon}} \cdot \gamma
$$

二者区别：

- LayerNorm 会减去均值，并除以标准差
- RMSNorm 不减均值，只根据均方根缩放
- RMSNorm 参数更少，计算更轻

RMSNorm 代码示意：

```python
class RMSNorm(nn.Module):
    def __init__(self, hidden_size, eps=1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(hidden_size))
        self.eps = eps

    def forward(self, x):
        # x: (B, T, hidden_size)
        rms = torch.rsqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return self.weight * x * rms
```

{IMPORTANT}归一化层不是为了改变序列长度或隐藏维度，而是为了让激活值分布更稳定，从而改善训练。{/IMPORTANT}

本节小结：Norm 层控制激活分布，提升训练稳定性，现代大模型常用 Pre-Norm 结构。

## 前馈网络 FFN：逐位置的非线性变换

{IMAGE:15}

Self-Attention 负责 token 之间的信息交互，而 Feed Forward Network 负责对每个位置的向量做非线性变换。

经典 FFN 结构：

$$
FFN(x) = W_2 \cdot \sigma(W_1x + b_1) + b_2
$$

一般会先升维再降维：

$$
d \rightarrow 4d \rightarrow d
$$

例如 hidden size 为 512，则中间层可能是 2048。

现代 LLaMA 类模型常用 SwiGLU 结构，而不是普通 ReLU FFN：

$$
FFN(x) = W_2(SiLU(W_1x) \odot W_3x)
$$

其中：

$$
SiLU(x) = x \cdot sigmoid(x)
$$

代码示意：

```python
class FeedForward(nn.Module):
    def __init__(self, hidden_size, intermediate_size):
        super().__init__()
        self.w1 = nn.Linear(hidden_size, intermediate_size, bias=False)
        self.w2 = nn.Linear(intermediate_size, hidden_size, bias=False)
        self.w3 = nn.Linear(hidden_size, intermediate_size, bias=False)

    def forward(self, x):
        # SwiGLU: 激活门控
        return self.w2(torch.nn.functional.silu(self.w1(x)) * self.w3(x))
```

{KNOWLEDGE}Attention 更像“混合不同 token 的信息”，FFN 更像“对每个 token 的内部表示做深层加工”。两者分工不同，缺一不可。{/KNOWLEDGE}

本节小结：FFN 为模型提供非线性表达能力，使每个 token 的表示经过更复杂的特征变换。

## 多层堆叠：从浅层特征到深层语义

{IMAGE:16}

一个 Transformer Block 的表达能力有限，所以实际模型会堆叠很多层。MiniMind 可能是较小规模模型，而大型语言模型可能有几十层甚至上百层。

堆叠形式：

```python
self.layers = nn.ModuleList([
    TransformerBlock(...) for _ in range(num_layers)
])
```

前向传播：

```python
for layer in self.layers:
    x = layer(x)
```

每一层都会进一步加工序列表示。一般可以粗略理解为：

- 浅层更关注局部词形、短距离关系
- 中层逐渐形成句法、语义关系
- 深层更接近任务相关和预测相关的表示

当然，这只是直观解释，实际神经网络内部表示并不完全按人工概念分层。

{WARNING}层数越多不一定越好。模型规模需要和数据量、训练算力、优化策略匹配。小数据训练过深模型，可能导致训练不稳定或过拟合。{/WARNING}

本节小结：Transformer 通过重复堆叠 Block 获得深层表达能力，模型容量主要由层数、隐藏维度、头数和 FFN 维度决定。

## 输出层：从隐藏状态到词表概率

{IMAGE:17}

经过所有 Transformer Block 后，模型得到最后的隐藏状态：

$$
H \in \mathbb{R}^{B \times T \times d}
$$

接着通过线性层映射到词表大小：

$$
logits = HW_{vocab}
$$

其中：

$$
W_{vocab} \in \mathbb{R}^{d \times V}
$$

因此输出形状为：

$$
(B, T, V)
$$

每个位置都会得到一个词表上的 logits 分布。

代码示意：

```python
class MiniMindModel(nn.Module):
    def __init__(self, vocab_size, hidden_size):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, hidden_size)
        self.lm_head = nn.Linear(hidden_size, vocab_size, bias=False)

    def forward(self, input_ids):
        x = self.embed(input_ids)

        # 这里省略多层 Transformer Block
        logits = self.lm_head(x)

        return logits
```

Softmax 后可以得到概率：

$$
P(x_{t+1}=i) = \frac{e^{z_i}}{\sum_{j=1}^{V}e^{z_j}}
$$

其中 $z_i$ 是第 $i$ 个词的 logit。

{IMPORTANT}训练时通常直接把 logits 交给 CrossEntropyLoss，不需要手动先做 softmax。PyTorch 的交叉熵内部已经包含 log-softmax。{/IMPORTANT}

本节小结：输出层把隐藏状态投影到词表维度，每个位置都会产生对下一个 token 的预测分布。

## 训练目标：Next Token Prediction

{IMAGE:18}

语言模型的训练目标是预测下一个 token。给定序列：

```text
我 喜欢 学习 大 模型
```

训练样本可以理解为：

```text
输入: 我 喜欢 学习 大
目标: 喜欢 学习 大 模型
```

也就是输入整体右移一位作为标签。

如果输入为：

$$
[x_1, x_2, x_3, ..., x_T]
$$

训练目标为：

$$
[x_2, x_3, x_4, ..., x_{T+1}]
$$

损失函数通常是交叉熵：

$$
Loss = - \sum_i y_i \log(p_i)
$$

其中：

- $y_i$ 是真实 token 的 one-hot 标签
- $p_i$ 是模型预测概率

PyTorch 示例：

```python
import torch
import torch.nn as nn

B, T, V = 2, 8, 6400

logits = torch.randn(B, T, V)
targets = torch.randint(0, V, (B, T))

loss_fn = nn.CrossEntropyLoss()

# CrossEntropyLoss 需要输入形状为 (N, C)
loss = loss_fn(
    logits.view(B * T, V),
    targets.view(B * T)
)

print(loss)
```

{WARNING}训练标签需要和输入错位对齐。如果输入和目标没有正确 shift，模型学到的预测关系会出错。{/WARNING}

本节小结：语言模型通过预测下一个 token 学习语言规律，训练损失通常是词表分类交叉熵。

## 推理生成：一个 token 一个 token 地续写

{IMAGE:19}

训练时可以并行计算整个序列的损失，但推理生成时通常是自回归的：

1. 输入 prompt
2. 模型输出下一个 token 的 logits
3. 选出一个 token
4. 拼回输入末尾
5. 重复生成

伪代码：

```python
def generate(model, input_ids, max_new_tokens):
    for _ in range(max_new_tokens):
        logits = model(input_ids)

        # 只取最后一个位置的预测
        next_token_logits = logits[:, -1, :]

        # 贪心选择概率最高的 token
        next_token = torch.argmax(next_token_logits, dim=-1, keepdim=True)

        # 拼接到输入序列后面
        input_ids = torch.cat([input_ids, next_token], dim=1)

    return input_ids
```

除了贪心搜索，还可以使用：

- temperature
- top-k sampling
- top-p sampling
- beam search

其中 temperature 控制分布平滑程度：

$$
p_i = softmax\left(\frac{z_i}{T}\right)
$$

当 $T$ 较小，输出更确定；当 $T$ 较大，输出更多样。

{KNOWLEDGE}训练和推理的数据流相似，但训练关心所有位置的 loss，推理只关心最后一个位置的下一个 token。{/KNOWLEDGE}

本节小结：推理阶段模型按自回归方式逐步生成，每一步都基于已有上下文预测下一个 token。

## 架构图中的数据形状流转

{IMAGE:20}

读 Transformer 图时，始终跟踪张量形状非常重要。以 batch size 为 $B$、序列长度为 $T$、隐藏维度为 $d$、词表大小为 $V$ 为例：

| 阶段 | 张量形状 | 含义 |
|---|---:|---|
| 输入 token ids | $(B, T)$ | 每个 token 的整数编号 |
| Token Embedding | $(B, T, d)$ | 每个 token 的向量表示 |
| 加入位置编码 | $(B, T, d)$ | 注入顺序信息 |
| Q/K/V 投影 | $(B, T, d)$ | 注意力三组表示 |
| 多头拆分 | $(B, h, T, d_h)$ | 多个注意力头 |
| Attention scores | $(B, h, T, T)$ | token 两两相关性 |
| Attention output | $(B, T, d)$ | 上下文融合结果 |
| FFN output | $(B, T, d)$ | 非线性变换结果 |
| Final logits | $(B, T, V)$ | 每个位置的词表预测 |

{IMPORTANT}学习 Transformer 时，最可靠的阅读方法是沿着数据流追踪 shape。只要 shape 能对上，代码实现通常就不会偏离架构太远。{/IMPORTANT}

本节小结：Transformer 架构图可以用形状流转串起来，从 $(B,T)$ 到 $(B,T,V)$ 是语言模型完整前向过程。

## MiniMind 实现中的核心模块映射

{IMAGE:5}

从架构图到 PyTorch 代码，通常可以拆成以下模块：

```python
class MiniMindLM(nn.Module):
    def __init__(self, config):
        super().__init__()

        # 1. token embedding
        self.tok_embeddings = nn.Embedding(
            config.vocab_size,
            config.hidden_size
        )

        # 2. 多层 Transformer Block
        self.layers = nn.ModuleList([
            TransformerBlock(config)
            for _ in range(config.num_hidden_layers)
        ])

        # 3. 最终归一化
        self.norm = RMSNorm(config.hidden_size)

        # 4. 词表输出层
        self.output = nn.Linear(
            config.hidden_size,
            config.vocab_size,
            bias=False
        )

    def forward(self, input_ids):
        x = self.tok_embeddings(input_ids)

        for layer in self.layers:
            x = layer(x)

        x = self.norm(x)
        logits = self.output(x)

        return logits
```

这个代码结构与架构图是一一对应的：

- Embedding 对应输入嵌入层
- `layers` 对应重复堆叠的 Transformer Block
- `Attention` 对应上下文信息交互
- `FeedForward` 对应非线性特征加工
- `RMSNorm` 对应训练稳定化
- `output` 对应词表预测

{KNOWLEDGE}从零实现大模型时，不要先追求复杂优化。先保证架构图中的主干路径清楚、shape 正确、loss 能下降，再逐步加入 KV Cache、混合精度、Flash Attention 等优化。{/KNOWLEDGE}

本节小结：MiniMind 的代码实现可以看作 Transformer 架构图的模块化翻译，每个图中方块都对应一个 PyTorch 类或函数。

## 常见易错点汇总

### 混淆 Encoder-Decoder 与 Decoder-only

经典论文中的 Transformer 包含 Encoder 和 Decoder，但 GPT 类大语言模型主要使用 Decoder-only。学习 MiniMind 时，应重点关注带因果遮罩的自注意力结构。

### 忘记 causal mask

没有 causal mask，模型训练时会看到未来 token，导致训练目标失真。

### logits 和概率混用

模型输出通常是 logits，不是概率。训练时直接输入 `CrossEntropyLoss`，推理采样时才需要 softmax 或相关采样策略。

### shape 变换错误

多头注意力中最容易出错的是：

```python
(B, T, C) -> (B, heads, T, head_dim)
```

以及最后拼接回：

```python
(B, heads, T, head_dim) -> (B, T, C)
```

### 残差连接维度不一致

残差相加要求两个张量形状完全一致，尤其 hidden size 不能随意改变。

本节小结：读懂架构图后，真正写代码时最容易出错的是 mask、shape、loss 对齐和 logits 处理。

## 关键结论

1. Transformer 的主线是：token ids → embedding → 多层 Transformer Block → logits。
2. Self-Attention 负责 token 间信息交互，FFN 负责逐 token 的非线性加工。
3. Decoder-only 模型必须使用 causal mask，保证预测时不能看到未来。
4. 多头注意力让模型从多个子空间理解上下文。
5. 残差连接和归一化是深层 Transformer 稳定训练的基础。
6. 训练目标是 next token prediction，本质上是对词表做分类。
7. 学习架构图时，要始终追踪张量形状，从 $(B,T)$ 到 $(B,T,V)$。
