## 课程背景

### 为什么要学习“从零实现”

{IMAGE:2}

现在很多人使用大模型时，主要通过 ChatGPT、Claude、通义千问、DeepSeek 等产品，或者通过 API 直接调用模型能力。这种方式适合应用开发，但如果想真正理解模型为什么能生成文本、为什么需要训练、为什么会出现幻觉、为什么上下文长度有限，仅会调用 API 是不够的。

从零实现一个小型语言模型，可以帮助我们建立三个层面的理解：

3. **工程层面**：理解数据处理、训练配置、显存消耗、模型保存与推理部署。

{KNOWLEDGE}大语言模型本质上是在学习一个条件概率分布：给定前面的 token，预测下一个 token。形式上可以写作：$P(x_t \mid x_1, x_2, ..., x_{t-1})$。{/KNOWLEDGE}

简要总结：从零手写不是重复造轮子，而是通过造一个“小轮子”理解真正的大系统。

## MiniMind 的学习路径

### 从语言模型的最小闭环开始

{IMAGE:3}

MiniMind 课程一般会围绕一个最小但完整的语言模型闭环展开：

数据文本输入后，先经过分词器变成 token id；token id 进入 Embedding 层变成向量；向量通过多层 Transformer Block；模型输出每个位置对词表中下一个 token 的预测 logits；再通过交叉熵损失计算误差；最后使用优化器反向传播更新参数。

可以把整个流程简化为：

$$
\text{text} \rightarrow \text{tokens} \rightarrow \text{embeddings} \rightarrow \text{Transformer} \rightarrow \text{logits} \rightarrow \text{loss}
$$

其中 logits 表示模型对词表中每一个 token 的未归一化分数。通过 softmax 可以将 logits 转换为概率：

$$
p_i = \frac{e^{z_i}}{\sum_j e^{z_j}}
$$

这里 $z_i$ 是第 $i$ 个 token 的 logit，$p_i$ 是它被预测为下一个 token 的概率。

{IMPORTANT}语言模型训练的核心目标：让真实下一个 token 的预测概率尽可能高。{/IMPORTANT}

简要总结：MiniMind 会围绕语言模型的完整闭环展开，从文本输入一直到模型输出和参数更新。

## PyTorch 在课程中的作用

### 用张量表达模型计算

{IMAGE:4}

PyTorch 是本课程的主要实现工具。它提供了张量计算、自动求导、神经网络模块、优化器和 GPU 加速能力。学习大模型时，PyTorch 不只是“写代码的框架”，更是把数学公式转化为可执行计算图的工具。

例如，最基本的语言模型训练步骤可以抽象为：

```python
import torch
import torch.nn as nn

# 假设 vocab_size 是词表大小，hidden_size 是隐藏层维度
vocab_size = 1000
hidden_size = 128

# 一个极简的 token embedding 层
embedding = nn.Embedding(vocab_size, hidden_size)

# 输入 token id，形状为 [batch_size, seq_len]
input_ids = torch.tensor([
    [12, 35, 78, 91],
    [44, 18, 23, 67]
])

# 输出向量，形状为 [batch_size, seq_len, hidden_size]
x = embedding(input_ids)

print(x.shape)
```

在真实模型中，`x` 还会继续进入 Transformer 模块，经过注意力机制和前馈网络处理。课程后续会逐步把这些模块从简单到复杂地搭建出来。

{WARNING}初学者很容易只关注代码是否能跑，却忽略张量形状。大模型代码中，很多错误都来自 batch、sequence length、hidden size、head size 等维度没有对齐。{/WARNING}

简要总结：PyTorch 是把大模型公式落到代码中的主要工具，理解张量形状是学习过程中的关键。

## 大模型的基本问题

### 模型到底在学什么

{IMAGE:5}

语言模型并不是直接“理解”一句话，而是通过大量文本学习词与词之间的统计关系。对于自回归语言模型来说，它的训练任务通常是 next token prediction，即预测下一个 token。

例如句子：

> 我 喜欢 学习 人工 智能

训练时可以构造多个预测任务：

- 输入“我”，预测“喜欢”
- 输入“我 喜欢”，预测“学习”
- 输入“我 喜欢 学习”，预测“人工”
- 输入“我 喜欢 学习 人工”，预测“智能”

数学上，一个长度为 $T$ 的序列 $x_1, x_2, ..., x_T$，模型训练目标可以写成最大化：

$$
P(x_1, x_2, ..., x_T) = \prod_{t=1}^{T} P(x_t \mid x_{<t})
$$

实际训练时通常最小化负对数似然，也就是交叉熵损失：

$$
\mathcal{L} = - \sum_{t=1}^{T} \log P(x_t \mid x_{<t})
$$

简要总结：语言模型通过预测下一个 token 学习语言规律，训练目标可以理解为让真实 token 的概率更高。

## Transformer 的核心地位

### 为什么现代大模型离不开 Transformer

{IMAGE:6}

MiniMind 虽然是小模型课程，但它的核心结构仍然会围绕 Transformer 展开。Transformer 是现代大语言模型的基础架构，GPT、LLaMA、Qwen、DeepSeek 等模型都与 Transformer 架构密切相关。

Transformer 的关键优势在于注意力机制。注意力机制允许模型在处理某个 token 时，根据上下文动态关注其他 token。例如在句子“苹果发布了新手机，它的价格很高”中，模型需要知道“它”更可能指代“新手机”，而不是“苹果”这个水果。注意力机制就是帮助模型建立这种上下文关联的核心结构。

简化的自注意力公式为：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

其中：

- $Q$ 是 Query，表示当前位置想查询什么信息。
- $K$ 是 Key，表示每个位置可以被匹配的特征。
- $V$ 是 Value，表示真正被聚合的信息。
- $d_k$ 是 Key 的维度，用于缩放点积，避免数值过大。

{KNOWLEDGE}注意力机制的本质是：根据相似度为上下文中的不同 token 分配权重，然后加权汇总信息。{/KNOWLEDGE}

简要总结：Transformer 是现代大模型的骨架，而自注意力机制是 Transformer 最核心的计算。

## 课程可能涉及的模块

### 从组件到完整模型

{IMAGE:7}

MiniMind 后续学习通常会拆分为多个模块，每个模块解决一个明确问题：

- **Tokenizer**：把文本切分成 token，并映射为整数 id。
- **Embedding**：把离散 token id 转换为连续向量。
- **Position Encoding**：让模型感知 token 的位置信息。
- **Attention**：建模上下文中 token 之间的关系。
- **MLP**：对每个位置的隐藏状态做非线性变换。
- **LayerNorm**：稳定训练过程。
- **Residual Connection**：缓解深层网络训练困难。
- **Loss Function**：衡量预测结果和真实答案的差距。
- **Optimizer**：根据梯度更新模型参数。
- **Generate**：根据模型输出逐步生成文本。

一个极简 Transformer Block 可以抽象成：

```python
class SimpleBlock(nn.Module):
    def __init__(self, attn, mlp, norm1, norm2):
        super().__init__()
        self.attn = attn
        self.mlp = mlp
        self.norm1 = norm1
        self.norm2 = norm2

    def forward(self, x):
        # 残差连接：保留原始信息，同时加入注意力输出
        x = x + self.attn(self.norm1(x))

        # 再经过前馈网络，并继续使用残差连接
        x = x + self.mlp(self.norm2(x))
        return x
```

这段代码体现了现代 Transformer 常见的 Pre-LN 结构：先归一化，再进入子模块，最后加残差。

简要总结：课程会把大模型拆成多个可理解、可实现的模块，最终拼成完整模型。

## 数据与训练的现实约束

### 为什么 MiniMind 要做“小模型”

{IMAGE:9}

真实工业级大模型训练成本极高，需要海量数据、多机多卡、复杂训练框架和大量工程优化。对于学习者来说，直接复现百亿参数模型并不现实。

MiniMind 的“小”有明确教学意义：

1. 参数规模小，便于本地运行和调试。
2. 代码结构清晰，方便逐行理解。
3. 训练成本低，可以快速看到实验结果。
4. 修改模块后能快速验证影响。
5. 不依赖复杂分布式训练，降低入门门槛。

{IMPORTANT}小模型不是低价值模型。对于学习原理来说，小模型更适合观察、调试和实验。{/IMPORTANT}

简要总结：MiniMind 通过缩小模型规模降低学习门槛，让学习者能真正掌控模型训练过程。

## 推理生成的基本直觉

### 训练和生成不是一回事

{IMAGE:11}

训练阶段，模型知道完整文本序列，可以并行计算每个位置的预测损失；生成阶段，模型只能根据已经生成的 token，一个接一个预测下一个 token。

生成流程可以简化为：

```python
def generate(model, input_ids, max_new_tokens):
    for _ in range(max_new_tokens):
        # 得到模型输出 logits
        logits = model(input_ids)

        # 只取最后一个位置的预测结果
        next_token_logits = logits[:, -1, :]

        # 转成概率分布
        probs = torch.softmax(next_token_logits, dim=-1)

        # 从概率分布中采样下一个 token
        next_token = torch.multinomial(probs, num_samples=1)

        # 拼接到已有序列后面
        input_ids = torch.cat([input_ids, next_token], dim=1)

    return input_ids
```

这里的关键点是：生成文本是一个循环过程。模型每次只预测一个 token，然后把这个 token 加回输入序列，继续预测下一个 token。

{KNOWLEDGE}训练更像“批改大量填空题”，生成更像“逐字续写文章”。两者使用同一个模型，但运行方式不同。{/KNOWLEDGE}

简要总结：语言模型训练和推理生成共享模型参数，但计算流程和使用场景不同。

## 常见误区

### 不要把大模型学习等同于背框架

{IMAGE:13}

很多初学者学习大模型时容易进入几个误区。

第一个误区是只看论文和公式，不写代码。这样容易觉得概念都懂，但真正实现时会卡在维度、mask、loss 对齐等细节上。

第二个误区是只调包，不理解内部机制。调用 `transformers` 库很方便，但如果不知道模型内部结构，就很难定位训练失败、显存爆炸或生成异常的原因。

第三个误区是追求大参数，而忽略可解释性。学习阶段更重要的是理解每个模块为什么存在，而不是一开始就训练更大的模型。

{WARNING}大模型工程里的很多问题不是“模型太小”，而是数据、训练目标、张量形状、mask 逻辑或推理策略出了问题。{/WARNING}

简要总结：课程应以理解和可控实验为主，避免陷入只背公式、只调 API、只追参数规模的误区。

## 一个极简语言模型示意

### 从结构上理解 MiniMind

{IMAGE:16}

下面是一个极简语言模型的示意代码，用于帮助理解后续课程可能展开的完整结构：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class TinyLanguageModel(nn.Module):
    def __init__(self, vocab_size, hidden_size):
        super().__init__()
        self.token_embedding = nn.Embedding(vocab_size, hidden_size)
        self.lm_head = nn.Linear(hidden_size, vocab_size)

    def forward(self, input_ids, labels=None):
        # input_ids: [batch_size, seq_len]
        x = self.token_embedding(input_ids)

        # logits: [batch_size, seq_len, vocab_size]
        logits = self.lm_head(x)

        loss = None
        if labels is not None:
            # 交叉熵要求输入为 [N, C]，标签为 [N]
            loss = F.cross_entropy(
                logits.view(-1, logits.size(-1)),
                labels.view(-1)
            )

        return logits, loss
```

这个模型还没有 Transformer，因此能力极弱，但它已经包含了语言模型最小训练闭环：输入 token、输出 logits、计算 loss。后续课程会逐步把中间的 Transformer Block 加进去，使模型具备上下文建模能力。

{IMPORTANT}复杂大模型可以看作是在这个最小闭环上不断增加表达能力、上下文能力和训练工程能力。{/IMPORTANT}

简要总结：极简模型帮助我们抓住主线，后续复杂结构都是在这个主线上的增强。
