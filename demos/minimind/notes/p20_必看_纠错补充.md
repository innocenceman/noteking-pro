## 第一部分：常见代码错误分类

{IMAGE:2}

### 1.1 数据类型相关错误

{KNOWLEDGE}PyTorch对数据类型极为敏感，错误的dtype会导致模型无法训练或梯度计算异常{/KNOWLEDGE}

在深度学习代码中，**数据类型不匹配**是最常见的错误之一。以下是典型问题：

```python
# ❌ 错误示例：dtype不一致
attention_scores = torch.matmul(query, key.transpose(-2, -1))
# 假设query是float32，key是float64，会导致精度问题

# ✅ 正确写法：统一使用相同的dtype
query = query.float()  # 确保与key一致
key = key.float()
attention_scores = torch.matmul(query, key.transpose(-2, -1))
```

{IMAGE:3}

### 1.2 张量形状错误

{IMPORTANT}矩阵运算必须严格遵循广播规则的维度要求{/IMPORTANT}

```python
# ❌ 错误：维度不匹配
# query: [batch, heads, seq_len, d_k]
# key:   [batch, heads, d_k, seq_len]
# 使用 transpose 时忘记调整正确维度
attention_scores = torch.matmul(query, key)  # 维度不兼容

# ✅ 正确：使用正确的transpose
attention_scores = torch.matmul(query, key.transpose(-2, -1))
# 结果维度: [batch, heads, seq_len, seq_len]
```

## 第二部分：注意力机制的错误修正

{IMAGE:4}

### 2.1 Softmax维度错误

注意力机制中的softmax必须作用在正确的维度上：

```python
# ❌ 错误：在错误维度上做softmax
attention_weights = torch.softmax(attention_scores, dim=-2)

# ✅ 正确：在key的维度上做softmax（通常是最后一个维度）
attention_weights = torch.softmax(attention_scores / math.sqrt(self.d_k), dim=-1)
# 对于 [batch, heads, seq_len_q, seq_len_k] 的张量
# softmax应该在 seq_len_k 维度上进行
```

{IMAGE:5}

### 2.2 Mask机制的正确实现

{KNOWLEDGE}掩码机制用于防止注意力看到当前位置之后的信息，是Transformer的核心组件{/KNOWLEDGE}

```python
def attention_mask(seq_len, device):
    """
    生成下三角注意力掩码
    """
    # ❌ 错误：使用torch.ones导致掩码逻辑错误
    mask = torch.ones(seq_len, seq_len, device=device)
    
    # ✅ 正确：生成下三角掩码
    mask = torch.tril(torch.ones(seq_len, seq_len, device=device))
    # 上三角部分为0（被mask掉），下三角部分为1（保留）
    
    return mask

# 在计算注意力时应用掩码
attention_scores = attention_scores.masked_fill(mask == 0, float('-inf'))
```

## 第三部分：模型参数初始化问题

{IMAGE:6}

### 3.1 权重初始化不当

{WARNING}不恰当的初始化会导致梯度消失或爆炸，严重影响模型收敛{/WARNING}

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads
        
        # ❌ 错误：直接使用nn.Linear，默认初始化可能不适合
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
        
        # ✅ 推荐：显式使用Xavier/He初始化
        for module in [self.W_q, self.W_k, self.W_v, self.W_o]:
            nn.init.xavier_uniform_(module.weight)
            if module.bias is not None:
                nn.init.zeros_(module.bias)
```

{IMAGE:7}

### 3.2 偏置项的处理

{IMPORTANT}在自注意力中，QKV变换的偏置通常设置为None以保持模型稳定性{/IMPORTANT}

```python
# ❌ 错误：为QKV变换添加不必要的偏置
self.W_q = nn.Linear(d_model, d_model, bias=True)
self.W_k = nn.Linear(d_model, d_model, bias=True)
self.W_v = nn.Linear(d_model, d_model, bias=True)

# ✅ 正确：移除偏置（现代Transformer的标准做法）
self.W_q = nn.Linear(d_model, d_model, bias=False)
self.W_k = nn.Linear(d_model, d_model, bias=False)
self.W_v = nn.Linear(d_model, d_model, bias=False)
```

## 第四部分：训练过程中的常见问题

{IMAGE:8}

### 4.1 梯度累积的正确实现

当batch_size受限时，需要使用梯度累积来模拟大batch训练：

```python
# ❌ 错误：每次都清零梯度
for batch in dataloader:
    optimizer.zero_grad()  # 错误：在每个batch开始时清零
    loss = model(batch)
    loss.backward()
    optimizer.step()

# ✅ 正确：使用梯度累积
accumulation_steps = 4
for step, batch in enumerate(dataloader):
    loss = model(batch) / accumulation_steps
    loss.backward()
    
    if (step + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()  # 只在累积足够梯度后清零
```

{IMAGE:9}

### 4.2 学习率调度

{IMAGE:10}

{KNOWLEDGE}学习率调度对大模型训练至关重要，预热(warmup)可以防止早期训练不稳定{/KNOWLEDGE}

```python
# ❌ 错误：使用固定学习率
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)

# ✅ 正确：使用学习率预热和余弦衰减
from torch.optim.lr_scheduler import CosineAnnealingWarmRestarts

scheduler = CosineAnnealingWarmRestarts(
    optimizer,
    T_0=10,           # 第一个周期
    T_mult=2,         # 后续周期倍数
    eta_min=1e-6      # 最小学习率
)

# 或使用线性预热
def lr_lambda(current_step, warmup_steps, total_steps):
    if current_step < warmup_steps:
        return float(current_step) / float(max(1, warmup_steps))
    return max(0.1, 1.0 - current_step / total_steps)
```

## 第五部分：数值稳定性问题

{IMAGE:11}

### 5.1 数值溢出与下溢

{IMPORTANT}指数函数在大负数时趋于0（数值下溢），大正数时趋于无穷（数值溢出）{/IMPORTANT}

```python
# ❌ 错误：直接计算指数，可能导致溢出
logits = torch.tensor([1000, 1001, 1002, 999])
probs = torch.exp(logits)  # 溢出！

# ✅ 正确：使用log_softmax或在softmax内部做数值稳定化
def stable_softmax(logits, dim=-1):
    # 减去最大值防止溢出
    max_logits = logits.max(dim=dim, keepdim=True).values
    exp_logits = torch.exp(logits - max_logits)
    return exp_logits / exp_logits.sum(dim=dim, keepdim=True)

# PyTorch内置的log_softmax更高效
log_probs = nn.functional.log_softmax(logits, dim=-1)
```

### 5.2 日志概率的计算

```python
# ❌ 错误：先转概率再取对数，浪费计算且不稳定
probs = torch.softmax(logits, dim=-1)
log_probs = torch.log(probs)

# ✅ 正确：直接使用log_softmax
log_probs = nn.functional.log_softmax(logits, dim=-1)
# 数学上等价: log_softmax(x) = x - log(sum(exp(x - max(x))))
```

## 第六部分：内存管理优化

{IMAGE:12}

### 6.1 梯度检查点（Gradient Checkpointing）

{WARNING}大模型训练必须考虑显存限制，合理使用梯度检查点可以大幅降低显存占用{/WARNING}

```python
# ❌ 错误：保存所有中间激活值，占用大量显存
class TransformerLayer(nn.Module):
    def forward(self, x):
        # PyTorch默认会保存所有中间激活值用于反向传播
        attn_output = self.attention(x)
        ff_output = self.feed_forward(attn_output)
        return ff_output

# ✅ 正确：使用梯度检查点，用计算换显存
from torch.utils.checkpoint import checkpoint

class TransformerLayer(nn.Module):
    def forward(self, x):
        # 使用checkpoint包裹，计算量增加约20-30%，但显存减少约50%
        attn_output = checkpoint(self.attention, x)
        ff_output = checkpoint(self.feed_forward, attn_output)
        return ff_output
```

### 6.2 混合精度训练

```python
# ❌ 错误：使用float32训练大模型
model = MiniMind()

# ✅ 正确：启用混合精度训练
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()
model = model.cuda()

for batch in dataloader:
    inputs = batch['input'].cuda()
    targets = batch['target'].cuda()
    
    with autocast():
        outputs = model(inputs)
        loss = criterion(outputs, targets)
    
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

## 总结

| 错误类型 | 关键修正点 |
|---------|-----------|
| 数据类型 | 确保所有张量使用统一的dtype |
| 维度错误 | 检查transpose和reshape的维度 |
| Softmax | 在正确维度上应用softmax |
| 初始化 | 使用适当的权重初始化方法 |
| 梯度累积 | 在累积步数达到后才清零梯度 |
| 数值稳定 | 减去最大值防止指数溢出 |
| 显存优化 | 使用梯度检查点和混合精度 |

## 本讲要点

{IMPORTANT}
1. **类型一致性**：所有参与运算的张量必须使用相同的dtype
2. **维度匹配**：矩阵乘法前务必检查张量形状
3. **数值稳定**：在指数运算前减去最大值
4. **显存优化**：大模型必须使用梯度检查点和混合精度
{/IMPORTANT}
