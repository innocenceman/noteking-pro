{IMPORTANT}**关键配置说明：**
- `batch_size`：增大可以提高梯度估计的稳定性，但会增加显存占用
- `learning_rate`：通常设置为 1e-3 到 1e-4，过高会导致训练不稳定，过低收敛太慢
- `gradient_clip`：防止梯度爆炸，通常设置为 1.0
- `mixed_precision`：使用 FP16 计算减少显存和加速训练{/IMPORTANT}

```python
scaler = GradScaler()
```

{KNOWLEDGE}**混合精度训练原理：**
混合精度训练使用半精度（FP16）进行前向和反向传播，同时在优化器状态中使用全精度（FP32）。这可以：
1. 减少 50% 的显存占用
2. 利用 Tensor Core 加速计算（如果有 NVIDIA Tensor Core GPU）
3. 基本不影响模型精度{/KNOWLEDGE}

### 2.1 训练循环框架

```python
import torch.nn.functional as F
from tqdm import tqdm
import time
import math

def train_epoch(model, train_loader, optimizer, scheduler, device, scaler, epoch):
    """
    单轮训练函数
    
    参数:
        model: 待训练的模型
        train_loader: 训练数据加载器
        optimizer: 优化器
        scheduler: 学习率调度器
        device: 计算设备
        scaler: 混合精度缩放器
        epoch: 当前轮次
    """
    model.train()  # 设置为训练模式
    total_loss = 0
    total_tokens = 0
    
    progress_bar = tqdm(train_loader, desc=f"Epoch {epoch}")
    
    for batch_idx, (inputs, targets) in enumerate(progress_bar):
        # 数据移至设备
        inputs = inputs.to(device)
        targets = targets.to(device)
        
        # ============ 核心训练步骤 ============
        
        # 1. 梯度清零（重要：避免累积）
        optimizer.zero_grad(set_to_none=True)
        
        # 2. 混合精度前向传播
        with autocast():
            outputs = model(inputs)
            loss = F.cross_entropy(
                outputs.view(-1, outputs.size(-1)),
                targets.view(-1),
                ignore_index=-100  # 忽略填充位置的损失
            )
        
        # 3. 缩放损失并反向传播
        scaler.scale(loss).backward()
        
        # 4. 梯度裁剪（防止梯度爆炸）
        scaler.unscale_(optimizer)
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        
        # 5. 参数更新
        scaler.step(optimizer)
        scaler.update()
        
        # 6. 学习率调度
        scheduler.step()
        
        # ============ 日志记录 ============
        
        # 计算已处理的 token 数
        batch_tokens = inputs.numel()
        total_tokens += batch_tokens
        total_loss += loss.item() * batch_tokens
        
        # 更新进度条
        if batch_idx % 10 == 0:
            avg_loss = total_loss / total_tokens
            lr = optimizer.param_groups[0]['lr']
            progress_bar.set_postfix({
                'loss': f'{avg_loss:.4f}',
                'lr': f'{lr:.2e}',
                'ppl': f'{math.exp(avg_loss):.2f}'
            })
    
    # 计算本轮平均损失
    avg_loss = total_loss / total_tokens
    perplexity = math.exp(avg_loss)  # 困惑度
    
    return avg_loss, perplexity
```

{IMPORTANT}**训练循环核心步骤总结：**

| 步骤 | 操作 | 目的 |
|------|------|------|
| 1 | `optimizer.zero_grad()` | 清空累积梯度，避免错误更新 |
| 2 | `loss.backward()` | 反向传播计算梯度 |
| 3 | `clip_grad_norm_()` | 梯度裁剪，防止梯度爆炸 |
| 4 | `optimizer.step()` | 更新模型参数 |
| 5 | `scheduler.step()` | 调整学习率 |

{/IMPORTANT}

### 2.2 前向传播与损失计算

{IMAGE:8}

```python
def forward_and_loss(model, inputs, targets):
    """
    前向传播与损失计算详解
    
    模型输入: [batch_size, seq_len] 的 token ID
    模型输出: [batch_size, seq_len, vocab_size] 的 logits
    """
    
    # Step 1: 模型前向传播
    # 输入形状: (B, L) → 输出形状: (B, L, V)
    logits = model(inputs)
    
    # Step 2: 计算语言模型交叉熵损失
    # 语言模型任务是预测下一个 token
    # logits: 预测的概率分布 (B*L, V)
    # targets: 真实 token ID (B*L,)
    
    loss = F.cross_entropy(
        logits.view(-1, logits.size(-1)),  # (B*L, V)
        targets.view(-1),                  # (B*L,)
        reduction='mean'
    )
    
    # Step 3: 可选：计算困惑度作为评估指标
    perplexity = torch.exp(loss)
    
    return loss, perplexity, logits
```

{KNOWLEDGE}**困惑度（Perplexity）解释：**
困惑度是语言模型最常用的评估指标，表示模型对测试数据的预测不确定性。

$$\text{PPL} = \exp\left(\frac{1}{N}\sum_{i=1}^{N}-\log P(x_i)\right)$$

- PPL = 1：完美预测
- PPL = 10：模型预测时平均有10个选择
- PPL 越低，模型越好{/KNOWLEDGE}

### 2.3 优化器配置

```python
from torch.optim import AdamW

def create_optimizer(model, config):
    """
    创建优化器，使用 AdamW 优化器
    
    AdamW = Adam + Weight Decay 修正
    """
    optimizer = AdamW(
        model.parameters(),
        lr=config['learning_rate'],
        betas=(0.9, 0.95),      # 动量参数
        eps=1e-8,               # 数值稳定性
        weight_decay=config['weight_decay'],
        fused=True              # 使用 fused kernel 加速（如果支持）
    )
    
    return optimizer
```

### 2.4 学习率调度策略

```python
from torch.optim.lr_scheduler import CosineAnnealingLR, LambdaLR

def create_scheduler(optimizer, config, num_training_steps):
    """
    创建学习率调度器
    
    常见策略：
    1. StepLR：固定步长衰减
    2. CosineAnnealingLR：余弦退火
    3. Linear Warmup + Cosine Decay：预热+余弦衰减（推荐）
    """
    
    # 策略1：预热+余弦衰减（GPT/BERT 等模型常用）
    warmup_steps = config['warmup_steps']
    
    def lr_lambda(current_step):
        if current_step < warmup_steps:
            # 线性预热：从 0 线性增长到初始学习率
            return float(current_step) / float(max(1, warmup_steps))
        else:
            # 余弦衰减：从初始学习率衰减到 0
            progress = float(current_step - warmup_steps) / \
                       float(max(1, num_training_steps - warmup_steps))
            return max(0.1, 0.5 * (1.0 + math.cos(math.pi * progress)))
    
    scheduler = LambdaLR(optimizer, lr_lambda)
    
    return scheduler
```

{WARNING}**学习率设置注意事项：**
1. 大模型通常使用较小的学习率（如 1e-4 到 1e-5）
2. Batch size 增大时，可按比例增大学习率（Linear Scaling Rule）
3. 预热阶段很重要，可以避免早期训练不稳定
4. 避免学习率过小导致陷入局部最优{/WARNING}

### 3.1 日志系统设计

```python
import logging
import json
from datetime import datetime

class TrainingLogger:
    """
    训练日志记录器
    记录训练过程中的关键指标
    """
    
    def __init__(self, log_dir):
        self.log_dir = log_dir
        self.metrics = {
            'train_loss': [],
            'train_ppl': [],
            'val_loss': [],
            'val_ppl': [],
            'learning_rate': [],
            'epoch_time': []
        }
        
        # 设置文件日志
        self.setup_logging()
    
    def setup_logging(self):
        """配置日志系统"""
        logging.basicConfig(
            level=logging.INFO,
            format='%(asctime)s - %(levelname)s - %(message)s',
            handlers=[
                logging.FileHandler(f'{self.log_dir}/training.log'),
                logging.StreamHandler()
            ]
        )
    
    def log_step(self, step, metrics):
        """记录每个训练步骤的指标"""
        if step % 10 == 0:  # 每10步记录一次
            logging.info(
                f"Step {step} | "
                f"Loss: {metrics['loss']:.4f} | "
                f"PPL: {metrics['ppl']:.2f} | "
                f"LR: {metrics['lr']:.2e}"
            )
            
            # 保存到内存
            self.metrics['train_loss'].append(metrics['loss'])
            self.metrics['learning_rate'].append(metrics['lr'])
    
    def log_epoch(self, epoch, train_metrics, val_metrics, epoch_time):
        """记录每个 epoch 的汇总指标"""
        log_msg = (
            f"\n{'='*60}\n"
            f"Epoch {epoch} Summary\n"
            f"{'='*60}\n"
            f"Train Loss: {train_metrics['loss']:.4f} | "
            f"Train PPL: {train_metrics['ppl']:.2f}\n"
            f"Val Loss: {val_metrics['loss']:.4f} | "
            f"Val PPL: {val_metrics['ppl']:.2f}\n"
            f"Epoch Time: {epoch_time:.2f}s\n"
            f"{'='*60}\n"
        )
        logging.info(log_msg)
        
        # 保存到文件
        self.save_metrics()
    
    def save_metrics(self):
        """保存指标到 JSON 文件"""
        with open(f'{self.log_dir}/metrics.json', 'w') as f:
            json.dump(self.metrics, f, indent=2)
```

### 3.2 关键训练指标解读

| 指标 | 计算公式 | 正常范围 | 异常信号 |
|------|----------|----------|----------|
| **Loss** | $L = -\frac{1}{N}\sum \log P(x_i)$ | 2.0 ~ 4.0 (收敛后) | Loss 上升或 NaN |
| **Perplexity** | $\text{PPL} = e^{\text{Loss}}$ | 7 ~ 55 (收敛后) | PPL 不下降 |
| **Learning Rate** | - | 随训练递减 | 学习率过小/过大 |
| **Gradient Norm** | $\lVert g \rVert_2$ | < 1.0 | 过大则爆炸 |

```python
# 梯度监控
def monitor_gradients(model):
    """监控梯度范数"""
    total_norm = 0.0
    for p in model.parameters():
        if p.grad is not None:
            param_norm = p.grad.detach().data.norm(2)
            total_norm += param_norm.item() ** 2
    total_norm = total_norm ** 0.5
    
    return total_norm
```

## 第四部分：完整训练脚本

```python
def main_training_pipeline():
    """
    完整的训练流程
    """
    # 1. 配置
    config = training_config
    
    # 2. 设备设置
    device = setup_device()
    
    # 3. 数据准备
    train_loader, val_loader = prepare_data(config)
    
    # 4. 模型创建
    model = MiniMindModel(config['model_config'])
    model = model.to(device)
    
    # 5. 优化器和调度器
    optimizer = create_optimizer(model, config['train_config'])
    num_training_steps = len(train_loader) * config['train_config']['num_epochs']
    scheduler = create_scheduler(optimizer, config['train_config'], num_training_steps)
    
    # 6. 混合精度缩放器
    scaler = GradScaler()
    
    # 7. 日志记录器
    logger = TrainingLogger('./logs')
    
    # 8. 训练循环
    best_val_ppl = float('inf')
    
    for epoch in range(1, config['train_config']['num_epochs'] + 1):
        start_time = time.time()
        
        # 训练
        train_loss, train_ppl = train_epoch(
            model, train_loader, optimizer, scheduler, 
            device, scaler, epoch
        )
        
        # 验证
        val_loss, val_ppl = evaluate(model, val_loader, device)
        
        epoch_time = time.time() - start_time
        
        # 记录日志
        logger.log_epoch(epoch, 
                        {'loss': train_loss, 'ppl': train_ppl},
                        {'loss': val_loss, 'ppl': val_ppl},
                        epoch_time)
        
        # 保存最佳模型
        if val_ppl < best_val_ppl:
            best_val_ppl = val_ppl
            save_checkpoint(model, optimizer, epoch, 'best_model.pt')
            logger.log.info(f"Saved best model with Val PPL: {val_ppl:.4f}")
    
    print("训练完成！")
    print(f"最佳验证困惑度: {best_val_ppl:.4f}")
```

### 5.1 梯度爆炸/消失

```python
# 问题诊断
if torch.isnan(loss):
    print("Loss is NaN! Checking gradients...")
    for name, param in model.named_parameters():
        if param.grad is not None:
            print(f"{name}: grad_norm={param.grad.norm():.4f}")
```

{WARNING}**梯度问题解决方案：**
- 梯度爆炸 → 降低学习率，增加梯度裁剪（max_norm=0.5）
- 梯度消失 → 检查残差连接，使用 Layer Norm，增加初始化方差
- Loss NaN → 检查数据归一化，减小学习率，使用混合精度{/WARNING}
